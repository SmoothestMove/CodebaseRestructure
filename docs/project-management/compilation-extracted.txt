2.  Core  Concepts  &  System  Architecture  2.1.  Fundamental  Principles  Smooth  Move  revolutionizes  the  moving  process  by:  
●  Digitizing  Inventory:  Moving  from  manual  lists  to  a  digital  tracking  system.  ●  Unique  Identifiers:  Assigning  a  unique  QR  code  and  human-readable  ID  to  every  
box.
 ●  Visual  Cues:  Utilizing  color-coding  for  quick  identification  of  ownership  and  room  
association.
 ●  Actionable  Workflow:  Guiding  users  through  distinct  stages  of  the  moving  
process
 
for
 
each
 
item.
 ●  Shared  Access:  Enabling  real-time  collaboration  among  multiple  users  involved  
in
 
the
 
same
 
move.
 2.2.  Key  Entities  2.2.1.  Owners  
●  Definition:  Users  can  define  distinct  "Owners"  for  belongings,  typically  individual  
family

 ●  Purpose:  Allows  personalized  management  of  items,  clear  differentiation,  and  
conflict-free

within
 
a

 ●  Attributes:  Each  Owner  is  assigned:  ○  A  unique  color  for  visual  identification  on  labels  and  in-app.  ○  Initials  for  their  Unique  Identifier  (UID)  prefix  (e.g.,  "JK"  for  John  Kepler).  ●  UID  Uniqueness:  The  system  ensures  that  Owner  initials  combined  with

numbers

in
 
unique

move
 
(e.g.,

Feature Name: Financial Navigator 
Purpose: To empower Smooth Move app users with comprehensive financial planning, 
tracking, and analysis capabilities for their self-managed moves, replicating the 
transparency and control a professional moving company might offer regarding costs. This 
feature aims to alleviate common budgeting pain points identified in market research, 
ensuring an efficient and effective user experience. 
 
1. Core Concepts & User Goals: 
• User Empowerment: Enable users to take full control of their moving finances. 
• Transparency: Provide clear visibility into estimated vs. actual spending. 
• Proactive Management: Allow users to identify potential overspending early and 
adjust. 
• Simplicity: Ensure ease of use for expense entry and budget management. 
• Collaboration: (Future consideration, if applicable for shared moves) Allow multiple 
users on a move to contribute to and view the budget. 
 
2. Key Functionalities & User Flows: 
2.1. Budget Setup & Management: 
• Functionality: Create and customize budget for a specific move. 
o User Flow: User navigates to "Financial Navigator" -> Initiates "Create New 
Budget" -> Enters a total estimated budget. 
o Developer Notes: Allow for a "default" set of categories, but enable users to 
add, edit, or remove custom categories. Each category should have an 
estimated budget field. 
• Functionality: Define/customize budget categories. 
o User Flow: User accesses budget settings -> Adds a new category (e.g., "New 
Appliances") or edits an existing one. 
o Developer Notes:  
▪ Initial Categories (Mandatory/Pre-populated):  
• Packing Supplies (boxes, tape, bubble wrap, markers) 
• Transportation (truck rental, gas, mileage, tolls, parking) 
• Professional Services (cleaning, handyman, appliance 
servicing, temporary labor) 

--- PAGE 18 ---
• New Home Essentials (initial groceries, cleaning supplies, 
utility setup fees/deposits) 
• Food & Refreshments (for moving day helpers/self) 
• Miscellaneous/Contingency 
▪ Customization: Enable users to define category names, and 
optionally, assign an icon or color for visual distinction. 
• Functionality: Set estimated budget per category. 
o User Flow: User enters an estimated monetary value for each active 
category. 
o Developer Notes: Ensure real-time calculation of total estimated budget as 
categories are filled. 
• Functionality (Future/Optional): Allocate expenses to specific "Owners" or 
"Communal Rooms" (linking to existing app entities). 
o User Flow: During expense entry, user can optionally tag an expense to an

o Developer Notes: This would require linking the budget module with the

2.2. Expense Tracking & Receipt Management: 
• Functionality: Smart Expense Entry with OCR Scanning (HIGH PRIORITY) 
o User Flow: User clicks "Add Expense" button -> User taps "Scan Receipt" -> 
Uses camera to take a photo of a receipt -> App processes image with OCR 
to extract amount, vendor, and date. -> User reviews and confirms extracted 
data, then selects category and adds optional description -> Saves. 
o Developer Notes:  
▪ Implement OCR API/SDK: Research and integrate a reliable OCR 
solution (e.g., Google Cloud Vision API, AWS Textract, or a specialized

▪ User Review & Correction: Present extracted data for user verification 
and allow for easy edits if OCR is inaccurate. 
▪ Fallback to Manual: If OCR fails or is not preferred, allow users to 
switch to manual input for all fields.

FK), box_qr_code (optional FK).

document into an interactive, dynamic, and personalized planning tool within the Smooth 
Move app. It will serve as a central hub for users to manage their entire moving process, 
from initial planning to post-move settlement, directly addressing the common challenge of 
users underestimating the time and organization required for a move. 
 
1. Core Concept: An Interactive Timeline & Task Manager 
The "Move Planner" will be a new, top-level feature accessible from the main dashboard. It 
will replace static, one-size-fits-all checklists with a dynamic timeline tailored to the user's 
specific move date. 
• Integration Point: A new "Move Planner" widget on the main dashboard will 
provide a snapshot of upcoming tasks. Tapping it will lead to the full feature page. 
• Key Functionality:  
o Timeline Generation: Upon first use, the app will ask for the user's planned 
moving date. Based on this date, it will automatically generate a dynamic, 
week-by-week timeline of tasks, using the "Timeline Development" section of 
the "Relocation Plans" as a blueprint. 
o Interactive Checklists: Each item on the timeline (e.g., "Research moving 
companies," "File change of address") will be an interactive task that the user 
can check off. 
o Customization: Users can add their own custom tasks, edit existing ones, 
and reassign tasks to different weeks, providing full control over their plan. 
o Smart Reminders: The app will leverage the timeline to send proactive 
notifications for upcoming deadlines (e.g., "Don't forget to book your movers 
this week!").

The true power of the "Move Planner" comes from its ability to connect with other parts of 
the Smooth Move app, creating a seamless and intelligent user experience. 
• Financial Navigator Integration:

Smooth Move App: Comprehensive Notification System 
Overall Goal: To provide timely, relevant, and actionable alerts and reminders that guide 
users through their move, enhance collaboration, and proactively mitigate stress, making 
the self-move experience as smooth as possible. 
Core Principles: 
• Contextual: Notifications are highly relevant to the current stage of the move or 
specific user actions. 
• Actionable: Where possible, notifications lead to a direct action within the app. 
• Customizable: Users have control over what, when, and how they are notified. 
• Concise: Messages are clear, brief, and easy to understand. 
• Non-intrusive: Balance informativeness with preventing notification fatigue. 
 
1. Notification Categories & Triggers: 
1.1. Planning & Setup Notifications: 
• Trigger: Move Creation/Upcoming Dates.  
o Notification: "Your move to [New Address] is scheduled for [Move Date]! 
Time to start planning!" (Welcome/Reminder) 
o Action: Directs to a "Move Checklist" or "Planning Dashboard." 
• Trigger: Nearing "Start Packing" date (based on user's move timeline).  
o Notification: "Packing time is almost here! Your estimated packing start 
date is [Date]. Have you ordered your supplies?" 
o Action: Link to "Packing Supplies Checklist" or Budget feature. 
• Trigger: Unassigned Inventory/Rooms after initial setup.

2. Notification Channels: 
• In-App Notifications: Displayed within the app (e.g., notification badge on an icon, 
toast messages, dedicated "Notifications" tab). Best for immediate feedback and 
less critical alerts. 
• Push Notifications: Sent to the user's device even when the app is closed. Ideal for 
time-sensitive, collaborative, or critical budget alerts. 
• Email Notifications: For important summaries, progress reports, or less urgent 
reminders that require more detail (e.g., weekly budget summary, pre-move 
checklist email). 
• SMS Notifications: Reserved for extremely critical, time-sensitive alerts (e.g., 
"Truck has arrived!" on moving day, if this level of integration is planned). Should be 
used sparingly. 
 
3. User Control & Preferences: 
• Granular Settings: Users should be able to enable/disable notifications for each 
category (e.g., "Packing Progress," "Collaboration Alerts," "Budget Warnings"). 
• Channel Preference: For each enabled category, users can select preferred 
channels (e.g., "Push," "Email," "In-App Only"). 
• Quiet Hours/Do Not Disturb: Option to set a period during which non-critical 
notifications are suppressed or delivered silently. 

--- PAGE 31 ---
• Sound/Vibration Settings: Standard OS-level controls. 
 
4. Technical Considerations: 
• Backend System: A robust notification service on the backend is required to 
manage triggers, user preferences, and dispatch notifications across channels. 
• API Integration: Integration with push notification services (e.g., Firebase Cloud 
Messaging, Apple Push Notification service), email service providers (e.g., 
SendGrid, Mailgun), and SMS gateways. 
• Database Schema: A notifications table to log sent notifications, 
user_notification_preferences table to store user choices. 
• Real-time vs. Batched: Determine which notifications need to be real-time (e.g., 
collaborator actions, critical budget alerts) vs. batched (e.g., daily/weekly 
summaries). 
• Localization: Support for different languages in notification messages. 
This comprehensive notification system aims to be highly effective by being relevant, 
timely, and user-controlled, ensuring the Smooth Move app truly serves as a personal 
logistics assistant.

Relocation Planning Framework 
 
I. Timeline Development 
Phase 1: Pre-Move Planning (8-12 Weeks Prior) 
• Week 8-12 Before Move: 
o Create a moving budget and set aside funds for unexpected expenses. 
o Research and get estimates from at least three moving companies, comparing 
services and pricing. 
o Begin decluttering and sorting belongings into keep, donate, sell, or discard 
categories. 
o Start researching your new neighborhood, schools, healthcare providers, and 
local services. 
o Create a dedicated moving binder or digital folder to organize all move-
related paperwork. 
• Week 6-8 Before Move: 
o Book your moving company or reserve rental trucks for DIY moves. 
o Begin collecting moving supplies including boxes, tape, bubble wrap, and 
packing materials. 
o Start packing non-essential items like seasonal decorations, books, and rarely 
used belongings. 
o Research utility providers in your new area and understand transfer 
procedures. 
o Begin gathering important documents and medical records for transfer. 
Phase 2: Packing and Logistics (4-8 Weeks Prior) 
• Week 4-6 Before Move: 
o File change of address with USPS (can be done online for $1.10). 
o Contact utility companies to schedule disconnection at old home and 
connection at new home. 
o Notify important parties of address change including banks, insurance 
companies, employers, and subscription services. 
o Continue systematic packing, focusing on rooms used less frequently. 
o Arrange time off work for moving day and notify employers of your 
relocation. 
• Week 2-4 Before Move: 
o Finalize moving day logistics and confirm details with your moving company. 

--- PAGE 33 ---
o Pack most belongings, leaving only essentials for the final weeks. 
o Arrange care for children and pets on moving day. 
o Start using up perishable food items and cleaning supplies. 
o Schedule cleaning services for your old home if needed. 
Phase 3: Moving Day (1-2 Weeks Prior & Day Of) 
• 1-2 Weeks Before: 
o Pack an essentials box with items needed for the first few days in your new 
home. 
o Confirm all utility transfers and ensure services will be active at the new 
home. 
o Disassemble furniture and prepare large items for moving. 
o Conduct final walkthrough of new home and note any pre-existing issues. 
o Get cash ready for tipping movers (typically $20-40 per mover for local 
moves, $40-80 for long-distance). 
• Moving Day: 
o Start early and be present to supervise the entire process. 
o Conduct final walkthrough of old home before movers finish loading. 
o Keep important documents, valuables, and essentials with you during the 
move. 
o Perform walk-through of new home upon arrival and direct placement of 
boxes and furniture. 
Phase 4: Post-Move Settlement (1-4 Weeks After) 
• Week 1 After Move: 
o Unpack essentials first: bedrooms, bathrooms, and kitchen basics. 
o Register to vote in your new location. 
o Find new healthcare providers, including doctors, dentists, and veterinarians. 
o Enroll children in new schools and arrange transportation. 
o Update driver's license and vehicle registration (typically required within 30 
days). 
• Weeks 2-4 After Move: 
o Complete unpacking and organizing your new home. 
o Explore your new neighborhood and locate essential services. 
o Register with new utility providers if not transferred. 
o Update voter registration and find local polling locations. 
o Leave reviews for your moving company and service providers.

III. Task Breakdown & Guides 
Packing Guide: 
Essential Packing Supplies: 
• Boxes in various sizes (small for heavy items like books, large for lightweight bulky 
items). 
• High-quality packing tape and tape dispenser. 
• Bubble wrap, packing paper, or newspaper for fragile items. 
• Permanent markers for labeling. 
• Packing peanuts or padding materials. 
Packing Strategies: 
• Use the "Russian Doll" technique: nest smaller packed boxes inside larger ones. 
• Wrap fragile items in clothing, linens, and towels instead of bubble wrap. 
• Pack heavy items in smaller boxes and lighter items in larger boxes. 
• Fill empty spaces with padding to prevent items from shifting. 
• Label boxes clearly with contents and destination room. 
• Color-code boxes by room for easy identification.
