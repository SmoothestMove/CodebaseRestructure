Project Overview
Build a modern, conversion-focused marketing website for Smooth Moves, a professional-grade logistics app for DIY movers. This is a landing page/marketing site that showcases the app and drives user adoption, NOT the app itself.
Brand Foundation (Critical - Read First)
Brand Identity

Purpose: Provide professional-grade logistics to DIY movers without professional-grade costs
Primary Feeling: Stress-free clarity about what needs to be done next
Voice: Calm, confident, directional - clarity beats personality
Personality: Friendly, playful, minimal, bold, plainspoken
Archetype: The Guide (reduce uncertainty through clear direction)

Visual Identity
Color System (use these exact roles):

Primary (Structure): #1E3A5F - Navigation, headers, section dividers only
Primary Action: #FF7E00 - ONLY for the single most important CTA per section
Secondary: #708090 - Supporting text, icons, inactive states
Accent: #E1A95F - Badges, subtle highlights only (use sparingly)
Neutrals: Background #D3D3D3, Surface #FFFFFF, Text #1F1F1F, Borders #C4C4C4

Typography:

Primary: Nunito (Regular 400, SemiBold 600, Bold 700)
Fallback: system-ui, -apple-system, Segoe UI, Roboto, sans-serif

Critical Rules:

Orange (#FF7E00) appears ONCE per screen section for primary action
Accent color is passive emphasis only, never for CTAs
Contrast and clarity trump visual flair
No jokes, emojis, or playful language in core messaging

Design Requirements
Modern Design Standards (2025)

Hero Section:

Full-viewport immersive hero with subtle parallax or scroll-triggered animations
Bold headline following "Outcome → How → Reassurance" formula
Single prominent CTA (orange) + one ghost/outline secondary CTA
Animated product mockup or interactive demo preview
Glassmorphism optional for overlay elements (frosted glass effect)

Visual Hierarchy:

Bento grid layouts for feature sections (modular, varied card sizes)
Generous white space - let content breathe
Sticky/floating navigation on scroll
Section transitions with smooth scroll animations
Micro-interactions on hover states (subtle, purposeful)

Contemporary Patterns:

Dark mode toggle (respect system preference by default)
Gradient overlays on images (subtle, brand-colored)
Card designs with soft shadows, not harsh borders
Animated statistics/counters on scroll into view
Testimonial carousel with auto-play and manual controls
FAQ accordion with smooth expand/collapse

Performance:

Mobile-first responsive (breakpoints: 640px, 768px, 1024px, 1280px)
Lazy loading for images below fold
Optimized animations (prefer CSS transforms over JS)
Fast initial paint - critical CSS inline

Technical Requirements
Tech Stack Recommendations

Framework: Next.js 14+ (App Router) or Astro (for performance)
Styling: Tailwind CSS (matches brand flexibility needs)
Animations: Framer Motion for complex interactions, CSS for simple ones
Icons: Lucide React (clean, modern, customizable)
Forms: React Hook Form + Zod validation
Deployment: Vercel or Netlify

Performance Targets

Lighthouse score: 95+ (all categories)
First Contentful Paint: < 1.5s
Time to Interactive: < 3.5s
Cumulative Layout Shift: < 0.1

Accessibility

WCAG 2.1 AA compliance minimum
Semantic HTML5
Keyboard navigation fully supported
Screen reader tested
Color contrast: 4.5:1 minimum for body text, 3:1 for large text

Animations Philosophy

Purposeful, not decorative: Every animation should guide attention or confirm action
Respect prefers-reduced-motion: Disable non-critical animations if user preference is set
Performance first: Use CSS transforms (translate, scale, opacity) over width/height
Timing: 200-300ms for micro-interactions, 400-600ms for section transitions
Easing: Use ease-out for entering, ease-in for exiting, ease-in-out for looping

Specific Animation Targets

Hero: Fade-in hero text (400ms), slide-up CTA (600ms delay)
Features: Cards fade/scale-up on scroll into view (stagger 100ms each)
Stats: Count-up animation when visible
Testimonials: Smooth crossfade transitions (every 5s, pause on hover)
Pricing cards: Hover lift effect (8px translate-y, 200ms)
CTA buttons: Scale pulse on hover (1.05x), active press (0.98x)

Copy Guidelines
Voice Rules (from brand bible)

Clarity beats personality—no clever wordplay
Show the next step—every section should guide action
Confident, not commanding—suggest, don't order
Specific over vague—avoid "easy," "simple," "quick"
Human, not casual—professional but warm
One action per message—don't stack CTAs

Headline Formula
Outcome → How → Reassurance
Good: "Track every box from pack to unpack. Real-time updates for your whole team. Zero learning curve."
Bad: "The easiest moving app ever! You'll love it!"
Forbidden Language

No emojis in primary copy (social links/testimonials okay)
No "just," "simply," "easy," "quick" without quantification
No questions as headlines unless immediately answered
No ALL CAPS except for acronyms
No exclamation points except in testimonials

Final Notes
Tone Check: If any section feels like it's "selling" too hard, dial it back. The brand is a confident guide, not a pushy salesperson. Every claim should feel matter-of-fact and verifiable.
Visual Balance: The site should feel calm and spacious, not busy. When in doubt, add white space. The orange CTA should be the only "loud" element per section.
Mobile Priority: Most users will browse on mobile. If a desktop layout doesn't gracefully collapse to mobile, redesign the desktop version.
Performance: Every animation and image should justify its file size. Compress ruthlessly. Smooth Moves promises to reduce stress—a slow website contradicts that promise.
Build this as if you're showing someone the way through a complex process, not selling them a product. That's the brand essence.
